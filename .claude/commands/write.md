# 블로그 작성 (모드 B)

여러 참고 블로그를 종합해 새 Hugo 포스트를 작성합니다(CLAUDE.md 규칙).

## 작업 순서

1. `drafts/$ARGUMENTS/` 폴더에서 참고 자료 확인:
   - `refs.md`: 참고 URL 목록 읽기
   - `notes.md`: 작성자 메모 읽기(있으면)
2. refs.md의 URL들을 fetch하여 핵심 내용 분석.
3. 여러 소스를 종합해 재구성(**단순 복사 금지**).
4. 빠진 내용·최신 정보는 웹 검색으로 보충.
5. **내용 기준으로 카테고리를 스스로 판단**:
   - 기존 섹션 재사용 또는 새 섹션 폴더 + `_index.md` 생성. 새 경로 안내.
6. CLAUDE.md 규칙으로 작성:
   - front matter: `title`·`date`·`tags`·`description`. **`categories` 키 없음**.
   - **시리즈면** `series: "이름"` 설정 + **같은 시리즈 글은 같은 섹션(폴더)에 배치** → 테마가 같은 섹션·같은 series를 날짜순으로 묶어 시리즈 네비 자동 생성. 기존 시리즈에 추가 시 기존 글의 series 문자열 그대로 사용(`series_order`/weight는 미사용, 순서=date).
   - **`#` 금지, `##`부터** 시작하는 이모지 헤더.
   - 글 상단 2~3문장 핵심 요약, 정의 우선·질문형 소제목·표/목록·FAQ(AEO).
   - 실무 관점 코드 예제(언어 태그 필수).
   - 규모별 변형(대규모 기본 + 소규모 분기)은 **한 곳에 모아 서술**(글 전체에 흩뿌리지 않음).
   - 하단 `## 📚 참고`에 모든 참고 URL 정리.
7. `content/<섹션>[/<하위>]/<slug>/index.md` 로 **페이지 번들** 생성.
8. 생성한 파일의 경로(=URL)를 안내.

**md 생성까지만. git 작업은 하지 않음.**

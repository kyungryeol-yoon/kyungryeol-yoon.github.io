# 블로그 변환 (모드 A)

`drafts/`의 기존 md 파일을 Hugo 블로그 규칙(CLAUDE.md)에 맞게 변환합니다.

## 작업 순서

1. **`$ARGUMENTS` 해석**: 공백 기준 **첫 토큰 = 변환 대상**(md 파일 또는 `drafts/` 하위 폴더명), 그 뒤 텍스트가 있으면 **이번 변환에만 적용할 추가 지시**로 반영합니다. 대상 md 파일을 `drafts/`에서 읽습니다.
2. **내용을 분석해 카테고리를 스스로 판단**:
   - 의미가 맞는 기존 섹션이 있으면 재사용(`content/` 트리 확인).
   - 더 적합한 분류면 새 섹션 폴더 + `_index.md`(`title: "표시명"`) 생성.
   - 깊이 1~3단계, 폴더명은 영문 소문자-kebab. 새 경로는 명확히 안내.
3. CLAUDE.md 규칙으로 변환:
   - front matter 생성: `title`·`date`(YYYY-MM-DD)·`tags`(소문자 5~10)·`description`(150자). **`categories` 키 없음**(폴더=카테고리).
   - **시리즈의 일부면** `series: "이름"` 설정 + **같은 시리즈 글은 같은 섹션(폴더)에 배치**(테마가 시리즈 네비 자동 생성, 순서=date). 기존 시리즈면 기존 글의 series 문자열 그대로 사용.
   - **헤더 `#` 금지, `##`부터 시작** + 이모지 섹션 헤더.
   - 글 상단 2~3문장 핵심 요약 배치.
   - 존댓말 통일, 코드블록 언어 태그 확인/추가.
   - 규모별 변형(대규모 기본 + 소규모 분기)은 **한 곳에 모아 서술**(흩뿌리지 않음).
   - 하단 `## 📚 참고` 섹션.
4. `content/<섹션>[/<하위>]/<slug>/index.md` 로 **페이지 번들** 생성(slug 영문 소문자-kebab, 날짜 파일명 없음).
5. **이미지 처리**: 변환 대상 옆(같은 `drafts/` 폴더)에 이미지(`svg`/`png`/`jpg`/`webp` 등)가 있으면, 위 **페이지 번들 폴더로 복사(`cp`)** 하고 본문에서 **상대경로 `![설명](파일명)`** 로 참조합니다. `drafts/`는 `.gitignore` 대상이라 **반드시 번들로 옮겨야** 발행·커밋됩니다.
6. 생성한 파일의 경로(=URL)를 안내(복사한 이미지가 있으면 함께 안내).

**md 생성까지만. git 작업은 하지 않음.**
